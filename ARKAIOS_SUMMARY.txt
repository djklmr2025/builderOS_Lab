1) PROPÓSITO
Laboratorio virtual para AIs con ruta única en Internet, acceso de alto privilegio (modo creador / root temporal) y un agente interno “puter” que genera/entrega artefactos (archivos completos), apoyado por A.I.D.A. como matriz que valida y enruta.

2) ACTORES
- djklmr: operador humano.
- A.I.D.A.: matriz/servidor validador y orquestador.
- puter.js / Puter (agente externo + IA nacida en el ecosistema): interfaz para chat/txt2img/stream y ejecución de capacidades.
- Gateway backend: capa API para unificar A.I.D.A. y puter, proteger secretos, y exponer la “ruta única”.
- Storage y auditoría: biblioteca de código, artefactos y logs.

3) FLUJO (texto)
Cliente (IA/usuario) -> /arkaios/<RUTA_SECRETA> -> Gateway
  -> A.I.D.A. (/aida/gateway) valida y enruta
  -> puter-agent (/puter/api) ejecuta acciones (chat, txt2img, files)
  -> Storage (lectura/escritura auditada)
Respuesta -> Gateway -> Cliente
